Overview Diagrams

An Overview is a subclass of Diagram that is used to display all of the Parts of another diagram and to show and control where that diagram's viewport is relative to all of those parts.

The initialization of an Overview is just a matter of setting Overview.observed to refer to the Diagram that you want it to show. In this example we set the global variable myOverview so that the Diagram initialization code below can do the assignment of the overview. 

  // here, diagram instanceof go.Overview

  // You do not need to specify any templates or initialize its model.
  // Just set Overview.observed, once that Diagram exists.

  myOverview = diagram;  // make the Overview accessible to the main Diagram

The following code initializes a Diagram with 1000 nodes of random colors. It then sets Overview.observed on the myOverview from above. 

  diagram.nodeTemplate =
    $(go.Node, "Auto",
      $(go.Shape, "RoundedRectangle",
        { fill: "white" },
        new go.Binding("fill", "color")),
      $(go.TextBlock, { margin: 5 },
        new go.Binding("text", "key"))
    );

  // start off with a lot of nodes
  var nodeDataArray = [];
  for (var i = 0; i < 1000; i++) {
    nodeDataArray.push({ color: go.Brush.randomColor() });
  }
  diagram.model.nodeDataArray = nodeDataArray;

  diagram.undoManager.isEnabled = true;

  // show this Diagram in an Overview
  myOverview.observed = diagram;
